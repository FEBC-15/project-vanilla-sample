# 3장 팀 프로젝트 생성
- 레포지토리는 멋쟁이 사자처럼의 Organizations 에서 생성
- [1. 프로젝트 생성](#1-프로젝트-생성)과 [2. 프로젝트 배포](#2-프로젝트-배포)는 각 프로젝트 팀별 디스코드 채널로 이동한 후 PL(Project Leader)이 화면을 공유해서 진행
- 팀원은 [3. 레포지토리 clone](#3-레포지토리-clone)만 진행

## 1. 프로젝트 생성
- 각 팀별 채널로 이동해서 진행

### 1.1 레포지토리 생성

- github 로그인
- Github organizations 접속
  - <https://github.com/FRONTENDBOOTCAMP-15th>
- Repositories > New repository
  - Repository name: vanilla-팀번호-프로젝트명(예시, vanilla-00-lionboard)
  - Description: 프로젝트 설명(예시, FrontEnd BootCamp 15기 바닐라 프로젝트 0팀 라이언보드)
  - **Public 선택**
  - Add a README file 체크
  - Create repository

### 1.2 Collaborators 추가

- Settings > Collaborators and teams > Add people
  - 팀원 계정 입력
  - Choose a role: Write
  - Add selection

### 1.3 개발환경 구축

- [2장 프로젝트 생성에서 배포까지 - 1.2 개발 환경 구축](./02.start.md#12-개발-환경-구축) 참고

## 3. 레포지토리 clone

- 팀원들은 위에서 생성하고 초기화가 완료된 레포지토리를 clone해서 프로젝트 시작

```sh
cd C:/febc15
git clone https://github.com/FRONTENDBOOTCAMP-15th/vanilla-팀번호-프로젝트명.git
```
