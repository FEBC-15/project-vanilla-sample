# 5장 깃허브 프로젝트

## 1. 깃허브 프로젝트란?
- 칸반 보드, 이슈/PR 연동 같은 기능을 제공하는 프로젝트 관리 도구

### 1.1 프로젝트 생성 실습
- Github Repository > Projects > New project > Table
  - Project name: `라이언 보드`
  - `Create project`

- 프로젝트에 팀원 추가
  - 생성한 프로젝트 > 우측 상단의 `...` > Settings > Manage access > Invite collaborators에 팀원 검색해서 `Invite`

### 1.2 뷰 이름 변경 실습
- 생성한 프로젝트 선택
- "View 1" 탭의 화살표 클릭 후 Rename view 선택
- View name: `백로그`
- `Save`

### 1.3 필드 추가 실습
- `백로그` 탭의 화살표 클릭 후 Fields 선택
- Labels, Milestone 체크 후 New field 선택
  + Field name: 우선순위
  + Field type: Single select
  + Options: `1차` 입력후 `Add` , `2차` 입력후 `Add`, `3차` 입력후 `Add`
  + `Save`
- `Save`

### 1.4 아이템 추가
- 프로젝트에 아이템을 단순히 메모 형태로 추가하거나(Create a draft) 이슈로 만들어서(Create new issue) 연동 가능
  + draft로 추가할 경우 title을 선택해서 Convert to issue 메뉴를 선택하면 이슈로 전환 가능
  + 이슈로 만들 때는 [ 4장 이슈 관리 - 3. 이슈로 지정하는 작업의 규모](./04.issue.md#3-이슈로-지정하는-작업의-규모) 참고

- 실습
  + 프로젝트 아이템 목록 맨 아래에 `You can use Control + Space to add an item` 누르면 프로젝트 아이템 추가 가능
  + `메인페이지 기본 마크업 작성` 입력 후 `Create a draft` 선택하거나 `Ctrl + Enter`
  + `회원 가입 페이지 기본 마크업 작성` 입력 후 `Create a draft` 선택하거나 `Ctrl + Enter`

### 1.5 칸반 보드 뷰 추가
- Todo, In Progress, Done 상태를 모아서 관리하는 칸반 보드 스타일의 뷰를 추가
- New View > Board
